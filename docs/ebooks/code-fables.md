## 第一章：控制流

### 条件分支

```python
if 努力 > 收获:
    return "继续"
else:
    return "调整"
```

人生充满 if-else。
关键不是选择哪条路，
而是理解为什么需要选择。

### 循环

```javascript
while (alive) {
  learn();
  grow();
}
```

人生不是 for 循环，
因为不知道何时结束。
但正因如此，
每一次迭代都有意义。

## 第二章：函数

### 参数

```python
def 人生(起点, 努力, 运气):
    终点 = 起点 + 努力 * 运气
    return 终点
```

参数不同，结果不同。
但函数本身是相同的——
你如何使用参数，决定了人生。

### 返回值

```go
func 意义() string {
    return "创造"
}
```

函数必须有返回值。
人生必须有意义。
不必在乎大小，
在乎的是真实。

## 第三章：并发

### 协程

```python
async def 工作():
    while True:
        await 做事()

async def 生活():
    while True:
        await 享受()
```

人生需要并发。
工作和生活不是串行——
并行运行，相互协作。

### 通道

```go
ch := make(chan 感悟)
go 倾听(ch)
ch <- 新的领悟
```

沟通是通道。
不是谁说得多，
而是信息是否有效传递。

## 第四章：错误处理

### 异常

```python
try:
    尝试(新事物)
except 失败:
    学习(失败)
    重试()
```

错误不是终点，
是学习的起点。
没有异常的人生，
才是真正的异常。

### 延迟

```go
defer 关闭(心门)
defer 记住(恩惠)
defer 放下(执念
```

延迟执行。
离开世界时，
才明白什么最重要。

## 第五章：并发安全

### 锁

```python
lock = Lock()
with lock:
    修改(状态)
```

人生需要锁——
关键时刻，专心致志。
不要同时做太多事，
除非你知道自己在做什么。

### 死锁

```go
// Thread A
mutexA.Lock()
mutexB.Lock()

// Thread B
mutexB.Lock()
mutexA.Lock()
```

两个线程互相等待，
永远无法前进。

人生亦然：
你等我，我等你，
最后谁也没动。

## 第六章：内存

### 指针

```c
int* 记忆 = &经历
*记忆 = 新领悟
```

记忆像指针。
指向过去的经历。
但你可以改变它的值——
赋予旧经历新意义。

### 垃圾回收

```python
del 执念
del 怨恨
gc.collect()
```

放下即回收。
不必记住所有事。
让记忆自动清理，
才能给新事物腾出空间。
